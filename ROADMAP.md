# 顾问决策系统 · 产品路线图与技术备忘

> 最后更新：2026-03-22

---

## 一、已完成功能（当前版本）

### 核心流程
- 首页输入决策问题，支持语音、图片、文件附件
- 智能质量信号（进度条提示输入是否充分）
- 诊断阶段：系统理解问题、激活相关顾问、生成追问
- 分析阶段：8位顾问流式输出，支持碰撞/共识检测
- 追问阶段：用户可针对具体顾问或整体继续对话
- 行动方案：AI生成结构化行动计划，支持新窗口打印/导出PDF

### 用户系统
- 姓名首次引导录入，统一存入 `user-settings`
- 设置页：档案、记忆与隐私、顾问偏好、分析方式、版本、数据管理
- 历史记录页：逐条删除 + 批量清空，自动过滤僵尸记录
- 数据全部存储在本地 `localStorage`，不上传服务器

### 体验优化
- 全局对比度提升（`--muted-foreground` 从 `#6B6560` → `#4A4540`）
- 清除所有多余 opacity 修饰符
- 历史记录入口、设置齿轮图标可见性改善
- Quick starters 三列 pill 布局
- 底部说明文字：体验版/专业版差异，数据存储说明

## 二、近期待完成

### 已设计、未接入的功能
- **设置中的「顾问数量」偏好**：`defaultAdvisorCount`（3/5/8）已存入设置，但诊断 API 返回的 `activatedAdvisors` 暂未受此影响，需要在 `/api/decisions/diagnose` 里读取该设置并截断或补充顾问列表

- **用户档案注入 prompt**：`occupation` 和 `concerns` 已有 UI，但尚未注入到顾问的 system prompt 里

## 三、画像系统（下一个重要里程碑）

### 背景与目标
让顾问真正"认识"用户，而不是每次从零开始。目标是用户感受到"这个系统越用越懂我"。

### 两个阶段

**第一阶段：画像摘要注入（优先做）**

在每次决策完成后，用轻量模型提炼本次决策的行为特征，追加更新 `user-profile`。每次分析时将画像摘要（200-300字）注入顾问 system prompt 开头。

画像包含六个维度：
1. **身份定位**：姓名、职业、人生阶段（"32岁，正在从职员向创业者转型"）
2. **决策风格**：分析型/直觉型、拖延/冲动、压力下保守/激进
3. **价值排序**：财务/关系/成长/稳定性的实际优先级（行为归纳，不是问卷）
4. **惯性陷阱**：反复出现的决策坑（"总用'再等等看'回避决断"）
5. **近期背景**：最近2-3次决策的领域和结论
6. **待验证假设**：用户自己说的判断，系统记录后与行为对照

存储结构：
```json
{
  "identity": "...",
  "decisionStyle": "...",
  "valueOrder": "...",
  "recurringTraps": "...",
  "recentContext": "...",
  "pendingHypotheses": "...",
  "updatedAt": "2026-03-22",
  "decisionCount": 3
}
```

存储位置：`localStorage` 的 `user-profile` 字段，与 `user-settings` 并列独立。

注意：`decisionCount < 3` 时，部分维度数据不足，注入 prompt 时跳过这些字段，不瞎猜。

**第二阶段：向量知识库（未来升级）**

当用户历史积累到20条以上时，画像摘要无法精准召回特定事件。需要引入向量数据库（如 Supabase pgvector），对历史决策做 embedding，每次分析时用语义相似度召回最相关的历史片段注入。

两个阶段的关系：画像摘要是基础层（从第1条记录就有价值），向量检索是精准层（历史体量大了才有意义）。不是两个方案，是同一件事的两个演进阶段。

### 启动条件
满足以下两个条件后再动手：
1. 有真实用户稳定使用，每人至少积累5条以上决策记录
2. 核心流程（首页→诊断→分析→追问→行动方案）体验已稳定，不再需要大改

### 当前占位处理
- 设置页"顾问参考历史决策"显示「即将上线」标签，不提供开关，避免用户误以为已生效
- 底部说明：数据存本地，换设备/清除浏览器后历史和画像将不再可用

## 四、数据存储结构总览

| 键名 | 内容 | 生命周期 |
|------|------|----------|
| `user-name` | 用户姓名（兼容旧版） | 持久 |
| `user-settings` | 用户设置（含姓名、偏好等） | 持久 |
| `user-profile` | AI归纳的决策画像（待实现） | 持久，累积更新 |
| `decision-history` | 历史记录索引列表 | 持久 |
| `decision-{id}` | 单条决策完整数据 | 持久 |
| `analysis-{id}` | 单条决策的分析状态 | 持久 |

---

## 五、已知局限与风险

1. **跨设备问题**：所有数据存本地，换设备后画像和历史清零。专业版上线时应考虑服务端同步
2. **隐私模式**：用户在无痕窗口使用时，localStorage 不持久，历史和画像每次会话结束即丢失
3. **体验版每日2次限制**：目前为硬编码UI提示，实际限速逻辑需要服务端配合
